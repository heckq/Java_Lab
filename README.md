# Звіт про програму для перевірки введення користувача

## Вступ

Ця програма створена для перевірки введення користувача. Вона включає власні класи виключень для обробки різних типів помилок введення, а також клас `InputValidator` для перевірки введення користувача.

## Класи виключень

Програма включає наступні класи виключень:

- `AgeOutOfRangeException`: Генерується, коли вік користувача не відповідає вимогам (від 18 до 100 років).
- `NameLengthException`: Генерується, коли довжина імені користувача не відповідає вимогам (від 2 до 100 символів).
- `InvalidNameException`: Генерується, коли ім'я користувача містить недопустимі символи (не букви).
- `DepositOutOfRangeException`: Генерується, коли ставка депозиту користувача не відповідає вимогам (від 0.1 до 100.0).

## Клас `InputValidator`

Клас `InputValidator` включає методи для отримання та перевірки введення користувача. Ці методи включають:

- `getValidAge()`: Перевіряє вік користувача.
- `getValidName()`: Перевіряє ім'я користувача.
- `getValidDepositRate()`: Перевіряє ставку депозиту користувача.

Якщо введення не відповідає вимогам, викликається відповідне виключення.

## Головна програма

Головна програма включає цикли `do-while` для кожного типу введення, яке потрібно перевірити. Якщо введення не відповідає вимогам, виводиться повідомлення про помилку, і програма повторно запитує введення. Цикл продовжується, поки користувач не введе правильне значення.

## Висновок

Ця програма демонструє, як можна використовувати власні класи виключень для обробки помилок введення. Вона також показує, як можна використовувати цикли для повторного запиту введення, поки користувач не введе правильне значення.
